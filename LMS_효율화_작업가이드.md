## 1. 브랜치 받아서 실행하기 (Windows CMD 기준)

```bat
cd C:\.claude\barunson_crm

REM 최신 브랜치 받기
git fetch origin claude/lms-campaign-efficiency-u28eoq
git checkout claude/lms-campaign-efficiency-u28eoq
```

## 2. 이번에 바뀐 것 & 테스트 체크리스트

캠페인 10개 발송 시 2시간+ 걸리던 수작업 병목 4곳을 자동화했습니다.

### ① 메시지 복제 개선  (탭: 캠페인 대시보드 → 메시지 작성)
- **이전 캠페인 메시지 불러오기** 드롭다운에서 하나 고르고 **[불러오기]**
  - ✅ 본문에 박혀있던 옛 `bit.ly/...` URL이 자동으로 **`{#URL}`로 복원**되는지
  - ✅ "이전 URL N개를 {#URL}로 자동 복원했습니다" 안내가 뜨는지
  - ✅ 발송일시가 비워져 새로 입력하게 되는지
- **변수 삽입 버튼**: `{#URL} 삽입` 외에 `{#이름}/{#A}/{#B}/{#C}/{#D}` 버튼으로 커서 위치에 삽입되는지

### ② A/B 발송양식 자동 2분할  (탭: 고객 추출)
- 조건 조회 후, **캠페인명 입력** → **[A/B 2분할 발송양식 다운로드]**
  - ✅ A파일(`..._A.xlsx`) / B파일(`..._B.xlsx`) **2개가 자동 다운로드**되는지
  - ✅ 대상자가 앞/뒤 절반으로 정확히 나뉘는지 (총 N명 → A=ceil(N/2), B=나머지)
  - ✅ 추출이력은 **"전체" 1건만** 저장되는지 (엑셀 수동분할 불필요)
  - 이후 A/B 캠페인 2건을 만들고 **각각 이 추출이력 + split(A/B)로 연동**하면 전환수가 그룹별로 자동집계됨

### ③ 추출 조건 프리셋  (탭: 고객 추출 → "조건 프리셋" 바)
- 필터를 세팅한 뒤 **[현재 조건 저장]** → 이름 입력
- 나중에 드롭다운에서 고르고 **[불러오기]** → 필터가 그대로 복원되는지
  - ✅ 라디오/날짜/최대건수까지 복원되는지, 날짜 하위칸 표시도 맞는지
  - ✅ **[삭제]** 동작
  - 저장은 서버(`extraction-presets.json`)라 브라우저/사용자 바뀌어도 유지됨

### ④ URL 생성·저장 원클릭 + 도달성 테스트  (탭: 캠페인 대시보드 → 발송 기록 / URL 관리)
- 캠페인 선택 → **[생성+저장 (원클릭)]** 한 번으로 Bitly 생성→발송기록 저장→`{#URL}` 치환까지
- **[도달성 테스트]**: 완성 URL로 실제 요청해 최종 상태코드/리다이렉트 확인
  - ✅ 정상 URL이면 "✔ 도달 OK (200)" 표시되는지
  - ⚠️ 사내망에서 barunsoncard.com 실도메인 도달 여부는 회사 아웃바운드 정책 영향

## 3. 아직 실DB로 최종 확인이 필요한 것

원격에선 사내 Azure SQL에 접속할 수 없어 아래는 **내 PC(실DB)에서** 확인 필요:
- 프리셋 불러온 뒤 **실제 조회 결과**가 조건대로 나오는지
- A/B 2분할의 **실제 명단** 분할·전환집계 정합
- 도달성 테스트가 사내망에서 실도메인에 닿는지
